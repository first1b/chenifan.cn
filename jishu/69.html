阿里云虚拟主机导出的数据出现T FOREIGN_KEY_CHECKS = 0;错误

未知 IT技术 发布时间：2020-06-10 0

阿里云有个虚拟机快到期了，今天把数据搬迁到ECS上了，但是在ECS上导入从阿里云虚拟机导出的数据时显示T FOREIGN_KEY_CHECKS = 0;错误，错误截图如下：

错误

SQL 查询：

T FOREIGN_KEY_CHECKS = 0;

MySQL 返回： 文档

#1064 - You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near 'T FOREIGN_KEY_CHECKS = 0' at line 1

解决方法：

打开从阿里云虚拟机导出的数据，删除第一行的SET FOREIGN_KEY_CHECKS = 0;代码。

删除这段代码重新导入即可。
